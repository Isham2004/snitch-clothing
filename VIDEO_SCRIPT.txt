================================================================
  SNITCH CLOTHING - VIDEO DEMONSTRATION SCRIPT
  Cross-platform fashion e-commerce app | Flutter + Firebase
  Target length: 3-5 minutes | Version 1.0.0
================================================================

HOW TO USE:
  "Quoted text" = what you SAY (voice-over / to camera)
  >> ACTION:    = what you DO on screen at that moment
  Aim for the middle of each time range to land ~4 minutes total.

BEFORE YOU RECORD:
  - Sign OUT of the app (so you can demo register/login fresh)
  - Have one test account ready (email + password) for fast login
  - Make sure the device is online so Firestore loads live
  - Open the Firebase Console in a browser tab (for Segment 7)
  - Test your screen recorder + microphone

================================================================
SEGMENT 1 - STUDENT INTRODUCTION                    [0:00 - 0:25]
================================================================
"Hi, my name is [YOUR NAME], student ID [YOUR ID]. This is my
 project demonstration for Snitch Clothing - a cross-platform
 fashion e-commerce mobile application I built using Flutter and
 Firebase. In the next few minutes I'll walk you through the
 app's main features and show it running live on a real device."

>> ACTION: Show your face or a title slide with the project
   name, your name, and student ID. Then cut to the device.


================================================================
SEGMENT 2 - APPLICATION OVERVIEW                    [0:25 - 1:00]
================================================================
"Snitch Clothing is a complete online fashion store. It's
 written in a single Flutter codebase that runs on Android, iOS,
 web, and desktop, and it uses Firebase for the backend -
 Firebase Authentication for accounts, Cloud Firestore for the
 database, and Cloud Storage for profile photos. The app covers
 the full shopping journey: sign up and log in, browse and
 search products, add items to a cart, check out, and track
 orders - all synced in real time across devices, with no
 custom server code."

>> ACTION: Launch the app so the animated splash plays. Show
   the Home screen - banner carousel, category strip, product
   grid - so the viewer sees the polished UI.


================================================================
SEGMENT 3 - LOGIN / REGISTER                        [1:00 - 1:40]
================================================================

[Register a new account]  (1:00 - 1:25)
"First, authentication. New users can register with an email and
 password. The form validates each field, and accepting the
 terms is required. When I submit, Firebase Authentication
 creates the account and the app writes a user profile document
 into Firestore."

>> ACTION: On the Register screen type a name, email, and
   password, toggle the password-visibility icon, tick the terms
   checkbox, tap Register. Show the app landing on Home.

[Login + reset + persistence]  (1:25 - 1:40)
"Existing users simply log in. If you forget your password, the
 'Forgot Password' dialog sends a reset email through Firebase.
 And because the session persists, the next time you open the
 app it skips the login screen entirely."

>> ACTION: Sign out, then sign back in with your test account.
   Open the 'Forgot Password' dialog briefly, then cancel.
   Optionally close/reopen the app to show it stays logged in.


================================================================
SEGMENT 4 - PRODUCT BROWSING                        [1:40 - 2:25]
================================================================

[Home, categories, search]  (1:40 - 2:05)
"All products load live from Firestore. The Home screen shows a
 banner carousel, featured products, and new arrivals. I can
 filter by category, and the search bar matches across product
 name, brand, category, and description."

>> ACTION: Scroll the Home screen. Tap a category chip to
   filter. Open search and type a query (product or brand name)
   to show live filtering.

[Shop screen + product detail]  (2:05 - 2:25)
"The Shop screen lets me sort by price or rating and switch
 between grid and list views. Opening a product shows a
 swipeable image gallery, the price with any sale discount, and
 selectors for size and colour before adding to the cart."

>> ACTION: Open the Shop tab, change the sort dropdown, toggle
   grid/list. Tap a product; swipe the image gallery and select
   a size and a colour.


================================================================
SEGMENT 5 - CART AND CHECKOUT                       [2:25 - 3:10]
================================================================

[Add to cart + manage cart]  (2:25 - 2:50)
"I'll add this item to the cart. The cart badge updates
 instantly. In the cart I can change quantities, swipe to delete
 an item, and the subtotal recalculates live - shipping is free
 over one hundred dollars."

>> ACTION: Tap 'Add to Cart' (show the snackbar). Open the Cart
   tab; tap +/- steppers, swipe one item to delete. Point out
   the updating subtotal.

[Checkout wizard]  (2:50 - 3:10)
"Checkout is a three-step wizard: delivery details, payment
 method, then a review of the full order. When I place the
 order, it's written to Firestore, the cart clears, and I'm
 taken straight to my order history."

>> ACTION: Tap 'Proceed to Checkout'. Walk through
   Delivery -> Payment -> Review, then tap Place Order. Show
   the success dialog.


================================================================
SEGMENT 6 - ORDER HISTORY                           [3:10 - 3:35]
================================================================
"The order I just placed appears immediately in my order history
 with a 'Confirmed' status. Each card expands to show the items,
 delivery address, payment method, and a tracking timeline.
 Because this is driven by a live Firestore stream, any status
 change from the backend would update here automatically."

>> ACTION: On the Orders screen, point to the new order. Tap to
   expand it and scroll through items, address, and tracking.


================================================================
SEGMENT 7 - FIREBASE INTEGRATION                    [3:35 - 4:20]
================================================================
"To prove this is all real and live, here's the Firebase Console
 for the project. In Authentication you can see the account I
 registered. In Cloud Firestore, the 'users' collection holds my
 profile, the 'carts' collection held my items, and the order I
 just placed is in both my private users/{uid}/orders
 subcollection and the top-level 'orders' collection. Profile
 photos go to Cloud Storage. Security rules make sure each user
 can only read and write their own data."

>> ACTION: Switch to the browser. Firebase Console ->
   Authentication (show the user). Firestore Database -> click
   through 'products', 'users', and 'orders' to show the
   document you just created. Optionally show the Rules tab and
   the Storage 'avatars/' folder.

   TIP: Have the order's Firestore document open and refresh it
   right after placing the order, so it appears live.

 browsing, a real-time cart and checkout, order history, and
 secure per-user data - all from a single codebase. Thank you

>> ACTION: Briefly show the physical device in frame (or the
   emulator window) to confirm it's live. End on the title slide
